## ⚠️ Reglas críticas del proyecto

### NO modificar
- **NUNCA modificar archivos .ejs de los clones SmartOLT** — inyectar JS vía middleware o modificando dashboard-smartolt-night.js
- **NUNCA reiniciar router borde** (CCR2116 10.0.0.1) — solo puente VPN WG + SSTP

### Infraestructura
- **Router core**: CCR2116 (10.50.255.245, SSH node-ssh admin/F1tfdrsx132022)
- **Router borde**: CCR2116 (10.0.0.1, admin/F1tfdrsx132022) — NUNCA reiniciar
- **hEX S**: 10.10.11.2
- **OLT ZTE C300 gestión**: 192.168.20.80 (zte/zte) vía SOCKS4 10.50.255.245:1080
- **SmartOLT VPN**: ovpn-out, IP 10.69.69.2
- **Proxy Telnet**: puerto 2323 + NAT #7 #8 en core

### Conexiones
- **fetch() a APIs ISPTotal**: usar `{ credentials: 'same-origin' }` siempre o da 401
- **MikroTik API**: puerto 8728 bloqueado en borde, usar SSH CLI: `nombre=valor`, `comment="..."`
- **SSTP**: usar workaround `disabled=yes` + `set disabled=no` para cambios

### Patrones de código
- **server.js monolito**: rutas fuera de app.listen() tienen prioridad sobre duplicadas dentro
- **SmartOLT authorize**: polling vía GET /api/smartolt/onu/authorize/progress
- **Render**: `renderPage(req, res, page, data)` con soporte standalone/layout
- **Auth**: middleware `requireAuth` verifica `req.session.user`

### Conexión OLT
- **GPONManager** usa `zte-olt.js` vía SOCKS4 (telnet-client)
- **SmartOLT** usa Puppeteer para clonar web UI
- **OLT persistente**: `olt-persistent.js` mantiene conexión abierta
- **ONU sync**: scripts en `/scripts/sync-onu-*.js` para sincronización batch

### Skills relacionados
- `isptotal-olt-management` — gestión OLT ZTE (descubrimiento, authorize, proxy Telnet)
- `isptotal-mikrotik-remote` — gestión remota MikroTiks cliente
